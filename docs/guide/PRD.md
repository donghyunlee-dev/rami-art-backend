## 📌 프로젝트 개요

### 배경 및 목적

라미아트미술교습소(ramiartstudio.com)는 현재 정적 웹사이트로 운영 중이며, 모든 콘텐츠(수업 정보, 갤러리, 블로그, 문의 내역 등)가 하드코딩되어 있습니다. 원장이 직접 콘텐츠를 수정하려면 개발자의 도움이 필요한 구조입니다.

본 프로젝트의 목표는 비개발자인 원장이 관리자 페이지를 통해 사이트의 모든 콘텐츠를 직접 관리할 수 있는 **CMS(Content Management System) 기반 관리자 시스템**을 구축하는 것입니다.

### 핵심 목표

- 원장(비개발자)이 로그인 후 콘텐츠를 CRUD할 수 있는 관리자 인터페이스 제공
- 관리자 변경사항이 실시간으로 프론트엔드에 반영
- 이미지 업로드/관리 기능 (Supabase Storage 활용)
- 문의 접수 및 관리 기능
- 단계별 개발로 MVP → 완성 순서로 구축

## 🚀 단계별 개발 계획

### Phase 기반 인프라 구축

**예상 기간: 1~2주**

- Supabase 프로젝트 생성 및 PostgreSQL 스키마 초기화
- Spring Boot 3 프로젝트 세팅 (Java 21, Gradle, JPA, Security)
- Supabase Auth 연동 및 JWT 기반 인증 구현
- 관리자 계정 초기 생성 (Supabase Auth)
- Next.js 프론트엔드 Supabase 클라이언트 연동
- 환경변수 관리 및 배포 파이프라인 설정

---

### Phase 관리자 로그인 및 기본 구조

**예상 기간: 1주**

- 사이트 헤더/푸터에 관리자 진입 링크 추가 (소형 텍스트 링크)
- `/admin/login` 관리자 로그인 페이지 구현
- JWT 토큰 기반 인증 및 보호된 라우트 미들웨어
- 관리자 대시보드 기본 레이아웃 (사이드바 메뉴 구조)
- Spring Boot Security 설정 및 `/api/admin/**` 경로 보호

---

### Phase 콘텐츠 관리 (핵심 CRUD)

**예상 기간: 3~4주**

- 홈 페이지 히어로 섹션 관리 (문구, 통계 수치)
- 수업 정보 관리 (유치부/초등부/중등부 CRUD)
- 갤러리 작품 관리 (이미지 업로드, 메타데이터)
- 블로그 게시글 관리 (WYSIWYG 에디터 포함)
- 소개 페이지 관리 (선생님 정보, 교육 철학)

---

### Phase 문의 관리 및 운영 기능

**예상 기간: 1~2주**

- 문의 접수 내역 조회 및 읽음/답변 상태 관리
- 위치/연락처 정보 관리
- 이미지 라이브러리 (Supabase Storage 연동)
- 사이트 기본 설정 관리 (메타태그, 운영 시간 등)

---

### Phase 고도화 및 QA

**예상 기간: 1~2주**

- 반응형 관리자 UI 최적화
- 이미지 최적화 (WebP 변환, 리사이징)
- 활동 로그 (관리자 액션 기록)
- 최종 QA 및 사용성 테스트
- 운영 배포 및 도메인 연결

## 🖥 관리자 화면 구성

### 진입 구조

사이트 푸터 또는 헤더에 **'관리자'** 링크(소형 텍스트)를 배치하여 `/admin/login`으로 이동합니다.
로그인 성공 시 `/admin/dashboard`로 리다이렉트됩니다.

```
[사이트] → 관리자 링크 클릭 → /admin/login → 인증 성공 → /admin/dashboard
```

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 로그인 | 이메일 + 비밀번호로 Supabase Auth 인증, JWT 발급 | P0 |
| 로그아웃 | 토큰 만료 및 세션 제거 | P0 |
| 토큰 갱신 | Refresh Token으로 자동 갱신 | P0 |
| 비밀번호 변경 | 관리자 설정에서 변경 가능 | P1 |
| 접근 제어 | `/admin/**` 경로는 인증된 관리자만 접근 | P0 |

## ⚡ 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 보안 | 관리자 API는 JWT Bearer 토큰 필수, HTTPS 강제 |
| 성능 | 관리자 페이지 초기 로드 3초 이내, 이미지 업로드 10MB 제한 |
| 가용성 | 프론트엔드(Vercel) 99.9% 업타임 |
| 확장성 | 추가 관리자 계정 등록 가능한 구조 |
| 접근성 | 관리자 UI는 데스크탑 최우선, 태블릿 지원 |
| 로깅 | Spring Boot Actuator + 관리자 액션 로그 |

## 🎯 기능 우선순위 정의

| 우선순위 | 정의 |
|----------|------|
| P0 | MVP 필수. 이것 없으면 서비스 불가. Phase 1~2에 완료. |
| P1 | 중요하지만 없어도 운영 가능. Phase 3~4에 완료. |
| P2 | 있으면 좋은 기능. Phase 5 이후 고려. |
